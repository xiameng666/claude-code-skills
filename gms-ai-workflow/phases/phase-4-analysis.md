# Phase 4: 业务梳理

## 目标

**叶子到根，分析业务功能，生成 MD 文档 + 标签**

## 前提条件

- Phase 3 已完成 (依赖图已构建)
- 所有类已发现

## 核心原则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  先理解业务，再还原代码                                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  当一个类的所有依赖都已梳理完成 → 才能梳理这个类                             │
│                                                                             │
│  依赖已梳理 = 下游类的业务功能已知 → 可以引用来解释当前类                    │
│                                                                             │
│  此阶段不生成 Java 代码，只做业务理解和文档                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ralph 循环

```bash
/ralph-loop "
  业务梳理任务

  输入: 依赖图 (SQLite)
  策略: 拓扑逆序 (叶子 → 根)

  单类流程:
  1. 选择: 依赖都已梳理的类 (叶子优先)
  2. 获取: 反编译代码 + 已重命名的符号
  3. 分析: 理解业务功能 (引用依赖类的功能说明)
  4. 标签: 识别类型、模块、功能标签
  5. 文档: 生成 MD
  6. 标记: analyzed=1

  注意: 此阶段不生成 Java 代码

  完成条件: 所有类梳理完成 + 文档生成
  完成时输出: <promise>ANALYSIS_COMPLETE</promise>
" --completion-promise "ANALYSIS_COMPLETE" --max-iterations 2000
```

## 单类业务梳理流程

```
┌─────────────────────────────────────────────────────────────────┐
│  单类业务梳理流程                                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 选择下一个可梳理的类                                         │
│     └─→ 查询: WHERE analyzed=0 AND 依赖都已 analyzed            │
│                                                                 │
│  2. 获取类信息                                                   │
│     └─→ get_class_decompiled_code()                             │
│     └─→ get_rename() (获取已重命名的符号)                       │
│     └─→ 读取依赖类的 MD 文档 (业务上下文)                        │
│                                                                 │
│  3. 业务分析                                                     │
│     └─→ 理解类的职责和业务逻辑                                   │
│     └─→ 分析与其他类的交互关系                                   │
│     └─→ 识别设计模式和架构角色                                   │
│     └─→ 引用依赖类的功能说明                                     │
│                                                                 │
│  4. 标签识别                                                     │
│     └─→ 识别类型标签 (proto, data_class, interface, etc.)       │
│     └─→ 识别模块标签 (location, fusion, auth, etc.)             │
│     └─→ 识别功能标签 (callback, handler, builder, etc.)         │
│                                                                 │
│  5. 生成 MD 文档                                                 │
│     └─→ frontmatter 包含标签                                     │
│     └─→ 记录业务功能                                             │
│     └─→ 引用依赖类的功能说明                                     │
│                                                                 │
│  6. 更新状态                                                     │
│     └─→ SQLite: analyzed=1, documented=1                        │
│     └─→ SQLite: tags 字段更新                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 标签系统

```
┌─────────────────────────────────────────────────────────────────┐
│  标签分类                                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  类型标签 (type):                                                │
│    • proto        - Protobuf 消息类                             │
│    • data_class   - 纯数据类 (只有字段和 getter/setter)          │
│    • interface    - 接口                                        │
│    • abstract     - 抽象类                                      │
│    • enum         - 枚举                                        │
│    • exception    - 异常类                                      │
│    • util         - 工具类                                      │
│                                                                 │
│  模块标签 (module):                                              │
│    • location     - 位置相关                                    │
│    • fusion       - 融合引擎                                    │
│    • auth         - 认证授权                                    │
│    • network      - 网络通信                                    │
│    • storage      - 数据存储                                    │
│                                                                 │
│  功能标签 (function):                                            │
│    • callback     - 回调接口                                    │
│    • handler      - 处理器                                      │
│    • builder      - 构建器模式                                  │
│    • factory      - 工厂模式                                    │
│    • singleton    - 单例模式                                    │
│    • listener     - 监听器                                      │
│    • manager      - 管理器                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## MD 文档模板

```markdown
---
tags:
  - proto
  - location
  - data_class
created_at: 2026-03-01
dependencies_analyzed: true
---

# {ClassName}

## 业务概述

**职责**: {一句话描述这个类的业务职责}

**业务逻辑**:
{详细描述，引用依赖类的功能说明}

**设计模式**: {如果识别出设计模式}

## 基本信息

| 属性 | 值 |
|------|-----|
| 混淆名 | {obfuscated_name} |
| 还原名 | {restored_name} |
| 父类 | {superclass} |
| 接口 | {interfaces} |
| 标签 | {tags} |

## 依赖关系

| 类 | 业务功能 |
|----|----------|
| {dep1} | {dep1的业务概述} |

## 字段

| 混淆名 | 还原名 | 类型 | 业务含义 |
|--------|--------|------|----------|
| {field} | {renamed} | {type} | {用途} |

## 方法

| 混淆名 | 还原名 | 签名 | 业务功能 |
|--------|--------|------|----------|
| {method} | {renamed} | {sig} | {职责} |

## 分析状态

- [x] 拓扑发现 (Phase 3)
- [x] 业务梳理 (Phase 4)
- [ ] Java 还原 (Phase 5)
```
